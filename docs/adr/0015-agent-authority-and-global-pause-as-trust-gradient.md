# Per-vendor `agent_authority` + global pause as the trust gradient

The agent's autonomy needs to be tunable per vendor and globally. Brand-new vendors should get less authority than vendors stealth has drafted well against for weeks; on sensitive days (audits, board meetings, traveling without laptop) the operator may want everything paused regardless of vendor. Boolean on/off is too coarse; per-action approval is too tedious; full autonomy from day one is too risky.

**Decision.** Each `Vendor` carries an `agent_authority` enum: `full_auto` (commits autonomously, downstream gates still apply), `draft_only` (drafts but never sends or marks state without explicit human approval — **default for new and seeded vendors**), `review_only` (observes and classifies only, doesn't draft). The agent reads this on every auto-fire activity. A platform-wide global pause flag (Redis-backed with TTL) suppresses all auto-fires regardless of per-vendor authority; toggled via admin header control or `/stealth pause Xm` Slack slash command. A bulk-promote admin action lets operators promote multiple vendors to `full_auto` once trust is established — turns the trust gradient into a 30-second action, not a per-vendor click marathon.
